# 笛 — 物理モデル管楽器

ウェーブガイド合成によるフルート音源の Web アプリ。管(bore)とジェット(jet)をそれぞれ遅延線としてシミュレートし、息の圧力がエッジのジェットを励起する。ジェットの非線形飽和と管の共鳴が相互作用して自励振動が育つ — ノイズ状の息が持続音へと立ち上がる。単声で、押している間だけ鳴る。

クリック / タップ、またはキーボード(`A S D F ...`)で吹鳴。

## 実行方法

ES Modules を使っているため、HTTP サーバー経由で開く(`file://` では動かない):

```sh
npx serve
# または
python3 -m http.server
```

ブラウザで表示されたポート(例 `http://localhost:8000`)を開く。ビルドは不要。

## 構成

```
index.html            マークアップ
style.css             スタイル
src/
  main.js             エントリポイント(初期化)
  config.js           定数(音名・スライダー仕様・キー割り当て)
  state.js            実行時状態の集約(周波数・パラメータ)
  ui.js               スライダー UI
  keyboard.js         画面上の鍵盤 + 物理キー入力
  audio/
    dsp.js            Flute — ウェーブガイドのシミュレーション本体(純粋・共有コード)
    flute-processor.js  AudioWorklet プロセッサ(Flute の薄いラッパ)
    engine.js         オーディオグラフ(コンプレッサ)と Flute へのメッセージ
```

音声合成は AudioWorklet(オーディオスレッド)で行い、worklet が使えない環境では同じ `Flute` を ScriptProcessorNode でフォールバック実行する。

## フルートのシミュレーション(audio/dsp.js)

- 管(`bore`)とジェット(`jetd`)の 2 本の遅延線を持つ。管の長さは `sr / freq` から決まり、ジェット長はその `jetRatio` 倍。
- 息の圧力(`maxP` × エンベロープ × ノイズ・ビブラート変調)がジェットを駆動し、`x·(x²−1)` の非線形飽和を経て管へ注入される。管端の反射はローパスと DC 除去を通す。
- 端数遅延は遅延線読み出し時の線形補間で扱うため、整数長に丸めずチューニングできる。

`dsp.js` は DOM / AudioContext 非依存の純粋モジュールなので、Node でそのまま数値検証できる。上位ディレクトリの撥弦楽器(絃 GEN)と同じ構成 — 純粋 DSP・worklet ラッパ・オーディオグラフの三層に分かれている。
